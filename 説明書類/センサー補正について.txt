センサーの精度と補正

センサー自体の精度については、説明書を参照ください。
精度については当方ではどうしようもないですが、
もし極端に悪い場合は交換いたします。(送料については負担ください。)

オフセットについて、
センサーの精度は概ね測定範囲の中心付近で誤差を最小にしてあると思います。
しかし、そのオフセットがずれている場合があるようです。
そこで、プログラムでオフセットを補正できるように補正係数を入力できるようにしています。
config.py に

def hosei():
    # センサーのオフセット誤差を補正する補正値です。
    temp  = 0
    humdi = 0
    press = 0
    return temp,humdi,press

変数があります。
ここに数値を入れることによりオフセットを変更できます。


<<センサーの誤差とオフセット考え方>>
実値に対して全ての値で測定誤差が+1度だとすると
-1度の補正をすれば、全ての値で誤差がゼロになりますが、

実際のセンサーは
20度で+1度、0度では+1.5度、60度では+2度の誤差があったりします。
補正を-1度とすると
20度で0度、0度では+0.5度、60度では+1度の誤差になります。

今回のセンサーは日常の気象条件の測定だと思いますので、
気温20度付近
湿度50%付近
気圧1005hpa付近
でなるべく信頼できる測定値と付き合わせて補正値を決定すれば良いと思われます。

ちなみに、気圧についてはスマホで所在地の気圧を確認するのが良いと思います。
スマホで得られる気圧値はどこかの観測点の計測値です。
その場所の標高とあなたの居る場所の標高が違うと数値が違ってきます。
厳密には温度も影響しますが、温度の影響は少ないです。
標高については10mで約1気圧違ってきます。
これを念頭にどの数字を表示するかを考えて補正値を使うかどうかを決定してください。
